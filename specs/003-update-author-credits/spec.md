# Feature Specification: 更新作者署名（增加开发时间与最后更新时间）

**Feature Branch**: `003-update-author-credits`
**Created**: 2026-03-11
**Status**: Done
**Input**: 启动时的文字后面，加注说明：软件作者：冯存华、邬巧玲，开始开发时间：2026-03-10 23:59:59，最后更新时间：当前时间（不是网页运行的当前时间）

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 查看开发者信息与时间戳（Priority: P1）

制作者希望在序幕页作者署名中，除了作者姓名外，还能展示两个静态时间戳：
- **开始开发时间**：固定为项目首次开发的日期时间
- **最后更新时间**：固定为本次代码修改并部署的日期时间

这两个时间均在**开发期间**手动写入 HTML，不在浏览器运行时动态生成。

**Why this priority**: 这是唯一的用户故事，属于最小可交付单元。时间戳让家人了解制作者为这份礼物所投入的心血与时间节点。

**Independent Test**: 直接打开 `index.html` 序幕页，目视确认作者署名区同时显示：作者姓名、开始开发时间和最后更新时间三行信息。

**Acceptance Scenarios**:

1. **Given** 用户打开网页序幕页，**When** 序幕动画播放完毕，**Then** 署名区显示「软件作者：冯存华、邬巧玲」
2. **Given** 用户打开网页序幕页，**When** 序幕动画播放完毕，**Then** 署名区显示「开始开发：2026-03-10 23:59:59」（固定不变）
3. **Given** 用户打开网页序幕页，**When** 序幕动画播放完毕，**Then** 署名区显示「最后更新：YYYY-MM-DD HH:MM:SS」，该时间为静态硬编码值，不随浏览器运行时变化
4. **Given** 签名文字存在，**When** 在手机或桌面浏览器查看，**Then** 三行信息均完整可读，字号小于主文字但不低于可读阈值

---

### Edge Cases

- **字段溢出**：时间戳字符较多，小屏幕（≤320px 宽）需换行显示，不得水平溢出
- **与现有动画冲突**：署名段落已使用 `.intro-line` 动画类，新增字段必须延续现有淡入序列，不跳出顺序

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**：署名区 MUST 显示作者姓名「冯存华、邬巧玲」
- **FR-002**：署名区 MUST 显示开始开发时间，格式为 `YYYY-MM-DD HH:MM:SS`，值固定为 `2026-03-10 23:59:59`
- **FR-003**：署名区 MUST 显示最后更新时间，格式为 `YYYY-MM-DD HH:MM:SS`，值为每次修改时手动更新的静态字符串（非 JS 运行时生成）
- **FR-004**：署名文字 MUST 参与 `.intro-line` 淡入动画序列，与现有动画顺序保持一致
- **FR-005**：署名字号 MUST 小于主文字（≤70%），颜色透明度低于主文字，视觉层级明确次要
- **FR-006**：小屏幕（≤320px）下，署名文字 MUST 自动换行，不产生水平滚动条

### Key Entities

- **IntroCredits（作者署名块）**：序幕页 `.intro-credits` 段落元素，包含三行内容（姓名、开始时间、更新时间），是纯静态 HTML 文本，无动态状态

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**：在 375px 宽手机视口中，署名三行信息均可完整显示，无截断或溢出
- **SC-002**：刷新页面任意次数，「最后更新」时间值始终保持不变（验证为静态值，非运行时生成）
- **SC-003**：署名字号视觉上明显小于序幕邀请信主文字，颜色偏灰偏透明，不影响主内容视觉焦点
- **SC-004**：目视验收通过率 100%（制作者本人确认显示正确后即视为完成）
