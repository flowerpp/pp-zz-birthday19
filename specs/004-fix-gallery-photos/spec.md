# Feature Specification: 修复记忆长廊滑动——仅第一张显示

**Feature Branch**: `004-fix-gallery-photos`
**Created**: 2026-03-11
**Status**: Draft
**Input**: bugfix: 运行 python3 -m http.server 8080，章节 2：记忆长廊的8张照片，只有第一张显示，后面都没显示出来

## 根因分析

长廊使用横向滑动展示8个场景。切换场景时，滑动容器（宽度为8个屏幕宽）通过 `translateX` 水平位移实现。当前位移量以**容器自身宽度的百分比**计算，而容器宽度是8个屏幕宽，导致每次切换实际位移量是预期值的8倍，场景2~8直接飞出可视区域，呈现为空白/不可见。

**预期行为**：点击下一个，滑动恰好一个屏幕宽度到达下一个场景。
**实际行为**：点击下一个，容器瞬间位移8个屏幕宽，所有内容消失不见。

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 正常浏览8个场景（Priority: P1）

用户点击「开始旅程」进入记忆长廊后，通过箭头按钮或滑动手势，依次查看8个人生阶段的照片和爸爸的话，每次切换都能流畅进入下一个场景。

**Why this priority**: 记忆长廊是本应用的核心叙事载体，8个场景必须全部可访问，否则超过87%的内容无法触达用户，整个礼物失去意义。

**Independent Test**: 打开网页 → 点击「开始旅程」→ 连续点击「›」按钮7次 → 确认每次都能看到对应场景的照片和文字。

**Acceptance Scenarios**:

1. **Given** 用户在记忆长廊第1个场景，**When** 点击右箭头「›」，**Then** 平滑滑动到第2个场景，场景标签显示「1岁」
2. **Given** 用户在第1个场景连续点击「›」7次，**When** 到达第8个场景，**Then** 显示「18岁」场景内容，右箭头隐藏，「继续 ✦」按钮出现
3. **Given** 用户在任意中间场景，**When** 点击左箭头「‹」，**Then** 滑动回上一个场景，场景标签编号减1
4. **Given** 用户在手机上，**When** 手指向左滑动超过50px，**Then** 切换到下一个场景

---

### User Story 2 - 进度指示器同步（Priority: P2）

用户每次切换场景时，底部进度指示器「X / 8」实时更新，与当前场景编号一致。

**Why this priority**: 进度指示器帮助用户了解自己在19年时光中的位置，是辅助信息而非核心功能。

**Independent Test**: 从场景1到场景8逐一切换，确认「1 / 8」→「2 / 8」→…→「8 / 8」同步变化。

**Acceptance Scenarios**:

1. **Given** 用户在第3个场景，**When** 点击「›」，**Then** 进度显示变为「4 / 8」
2. **Given** 用户在第8个场景，**When** 进度显示为「8 / 8」，**Then** 「›」箭头隐藏，「继续 ✦」按钮可见

---

### Edge Cases

- **快速连续点击**：连续快速点击箭头时，动画锁应防止场景跳跃或错位
- **横屏模式**：手机横屏时，场景布局切换为横排（照片+文字左右排列），滑动行为不变

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**：每次场景切换，滑动容器的水平位移量 MUST 恰好等于一个屏幕宽度，不多不少
- **FR-002**：场景切换动画 MUST 保持平滑过渡效果，无抖动、无跳变
- **FR-003**：8个场景 MUST 均可通过右箭头按钮或左滑手势访问
- **FR-004**：切换过程中（动画进行时），MUST 锁定导航防止重复触发
- **FR-005**：场景切换后，进度指示器、导航按钮可见性 MUST 随之同步更新

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**：从第1个场景连续点击「›」7次，能依次看到全部8个场景，成功率100%
- **SC-002**：场景切换视觉上无跳变——滑动幅度恰好覆盖一个屏幕宽度
- **SC-003**：手机端（iOS Safari / Android Chrome）滑动手势正常切换场景，与桌面端行为一致
- **SC-004**：修复后，`python3 -m http.server 8080` 本地验收通过，无需任何额外配置
