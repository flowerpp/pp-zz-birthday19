# Feature Specification: 修复记忆长廊场景 6~8 照片不显示

**Feature Branch**: `005-fix-gallery-scenes-6-8`
**Created**: 2026-03-11
**Status**: Draft
**Input**: bugfix: 运行 python3 -m http.server 8080，章节 2：记忆长廊的8张照片，后面的6-8都没显示出来

## 根因分析

**照片文件均存在**，问题在于图片加载策略：

场景照片使用了「懒加载」（lazy loading）机制——浏览器只加载距离当前视口足够近的图片。但长廊容器设有 `overflow: hidden`，且场景的位置偏移通过 CSS 变形（transform）实现，**浏览器以元素的布局位置（非视觉渲染位置）判断是否触发懒加载**。

场景 6~8 的图片在页面布局中距视口超过 500~700 个屏幕宽度，远超浏览器懒加载阈值（通常 1250~2500px），因此这些图片从未被加载——即使用户已滑动到该场景，CSS transform 让画面看起来已到位，但图片始终处于「待加载」状态。

**根因**：`js/scenes.js` 中对场景照片设置了 `loading="lazy"`，与长廊的 CSS transform 滑动模式不兼容。

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 场景 6~8 照片完整显示（Priority: P1）

用户滑动到第 6、7、8 个场景时，对应的照片能够正常显示，不出现空白、破损图标或仅显示文字而无图片的情况。

**Why this priority**: 场景 6（12岁·毕业舞姿）、7（16-17岁·疯玩花季）、8（18岁·首医朝圣）是时间线的后半段，承载了成长中最重要的青春记忆，若无法显示则整个故事叙事残缺。

**Independent Test**: 打开网页 → 点击「开始旅程」→ 连续点击「›」5次到达场景6 → 确认场景6、7、8的照片均正常显示，不需要刷新页面。

**Acceptance Scenarios**:

1. **Given** 用户已进入记忆长廊，**When** 点击「›」切换到第6个场景（12岁），**Then** 场景内的3张照片全部正常显示，无空白区域
2. **Given** 用户已进入记忆长廊，**When** 点击「›」切换到第7个场景（16-17岁），**Then** 场景内的3张照片全部正常显示
3. **Given** 用户已进入记忆长廊，**When** 点击「›」切换到第8个场景（18岁），**Then** 场景内的3张照片全部正常显示
4. **Given** 任意场景，**When** 页面首次加载后无需任何操作，**Then** 导航到场景6~8时照片立即可见，无需等待加载

---

### Edge Cases

- **弱网环境**：即使在弱网下，场景6~8的照片也应在进入场景前完成加载（或至少开始加载），不出现长时间空白
- **场景1~5**：修复后，场景1~5的照片加载行为不得退步，仍应正常显示

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**：记忆长廊全部8个场景的照片 MUST 在用户导航到该场景时能够显示，不得出现空白或破损图标
- **FR-002**：场景6、7、8的照片 MUST 在页面加载后（进入长廊之前）即开始加载，确保导航到时可即时显示
- **FR-003**：修复 MUST 不影响场景1~5已正常工作的照片显示行为
- **FR-004**：修复 MUST 不引入新的网络请求延迟或页面性能明显下降（照片总数仅约24张，全量加载可接受）

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**：从场景1逐一切换到场景8，全部8个场景的照片均能正常显示，成功率100%
- **SC-002**：场景6~8在本地 `python3 -m http.server 8080` 验收中无空白图片
- **SC-003**：修复后场景1~5的显示行为与修复前一致，无退步
