# 功能规格：《给19岁的你》— 壮壮生日互动叙事网页游戏

**功能分支**: `001-birthday-memory-game`
**创建日期**: 2026-03-11
**状态**: 草稿
**输入**: 为19岁女儿壮壮制作网页版互动叙事生日礼物游戏，部署至 GitHub Pages

---

## 用户场景与测试 *(必填)*

### 用户故事 1 — 序幕：推开时光之门 (优先级: P1)

壮壮在生日当天收到链接，用手机打开网页，看到水彩风格的封面动画缓缓呈现——
一封手写风格的信浮现在星空之上，邀请她"推开这扇门，走进你的19年"。
她点击"开始旅程"按钮，音乐轻轻响起（生日快乐歌）。

**为什么优先**: 这是整个体验的入口，决定第一印象和情感连接。

**独立测试**: 单独访问首页即可测试——在手机浏览器中打开后能看到完整封面动画和开始按钮。

**验收场景**:

1. **给定** 用户首次访问网页，**当** 页面加载完成，**则** 出现星空水彩背景、手写风格邀请信文字、"开始旅程"按钮，背景音乐自动或点击后播放。
2. **给定** 用户点击"开始旅程"，**当** 点击触发，**则** 以平滑动画过渡至第一个记忆场景。
3. **给定** 用户在手机竖屏模式，**当** 页面渲染，**则** 所有元素正确自适应显示，无需横屏即可体验完整内容。

---

### 用户故事 2 — 记忆长廊：8个人生阶段场景 (优先级: P1)

壮壮通过滑动（手机）或点击箭头（PC），漫步穿过8个记忆场景，每个场景展示：
- 该年龄段的精选照片（相框或翻页效果）
- 爸爸手写风格的一句话描述
- 点击照片可触发小动画彩蛋

**8个场景内容**：

| 场景 | 年龄 | 爸爸的话 |
|------|------|---------|
| 1 | 0岁 | 2007年3月11日11点03分，一个女孩来到这个世界，6斤重。你被推出产房那一刻，世界突然变得非常温柔。感谢老婆！ |
| 2 | 1岁 | 各种和我们互动，学会了使用工具，学会了各种搞怪！ |
| 3 | 5岁 | 参加儿童组全国歌唱大赛，大气参赛、捧回了金奖，却不参加领奖，高冷！ |
| 4 | 10岁 | 我们一家去张北旅行，酷飒！ |
| 5 | 11岁 | 我们一起去日本，东京大学我们碰到了一个小哥哥。 |
| 6 | 12岁 | 你小学毕业，舞蹈的身姿照片至今应该还在和4的楼道墙壁上！ |
| 7 | 16-17岁 | 花季年龄，虽然经历了新冠和即将到来的高考，但是我们还是到处疯玩——杭州、海南、武汉、欧洲、联合国！ |
| 8 | 18岁 | 壮壮考上了首医心仪的专业，我们也去了五台山！ |

**为什么优先**: 这是游戏的核心情感体验，所有珍贵记忆在此展现。

**独立测试**: 可逐个场景独立验证——每个场景在任意起始点均可完整展示照片、文字和动画。

**验收场景**:

1. **给定** 用户进入记忆长廊，**当** 向左滑动或点击下一步，**则** 以平滑动画切换到下一个场景。
2. **给定** 用户到达任意场景，**当** 场景加载，**则** 展示该年龄段对应的照片（至少1张）、爸爸的话、当前进度指示（如"3 / 8"）。
3. **给定** 用户点击场景中的照片，**当** 点击触发，**则** 播放对应彩蛋动画（如星光粒子、文字浮现、爱心飘落）。
4. **给定** 用户在最后一个场景（18岁），**当** 点击"继续"，**则** 过渡至19朵星光界面。

---

### 用户故事 3 — 19朵星光：祝福时刻 (优先级: P2)

穿越8个记忆场景后，长廊尽头天空飘落19颗星星。
壮壮逐一点开每颗星，每颗星里藏着爸爸妈妈写给她的一句祝福或愿望。

**19句祝福内容**：

1. 壮壮，姑且20岁之前我们还这么叫你，爸爸妈妈永远爱你。
2. 亲爱的女儿，19岁意味着新的开始。
3. 未来的路很长，但无论你去哪里，爸爸妈妈永远是你最坚定的支持者。
4. 永远勇敢，永远热爱自己。
5. 愿你始终保持那份大气从容——就像5岁拿了金奖却不领奖的你。
6. 愿你的笑容永远那么灿烂，温暖每一个靠近你的人。
7. 愿你在医学的路上，用你的善良治愈世界。
8. 愿你永远保持对世界的好奇心，就像第一次去日本时的你。
9. 愿你走过更多的山川大海，带回更多的故事。
10. 愿你永远有爱你的人在身旁。
11. 愿你健康快乐，永远壮壮。
13. 愿你永远是那个敢于高冷、敢于特立独行的自己。
18. 十九年，谢谢你让我们成为你的爸爸妈妈。
19. 生日快乐，壮壮，我们爱你。❤️

**为什么优先**: 这是情感高潮，是整个体验的升华时刻。

**独立测试**: 可直接跳转至星光界面，逐一点击19颗星验证展示效果。

**验收场景**:

1. **给定** 用户完成8个场景旅程，**当** 进入终幕，**则** 19颗星星以动画效果依次出现在星空背景上。
2. **给定** 用户点击任意一颗星，**当** 点击触发，**则** 以优美动画展开该星对应的祝福文字，文字清晰可读。
3. **给定** 用户已点开所有19颗星，**当** 最后一颗星打开，**则** 触发终幕彩蛋。

---

### 用户故事 4 — 终幕彩蛋：只属于我们的话 (优先级: P2)

第19颗星点开时，触发特别彩蛋：精选家庭照片以光晕效果缓缓呈现，
配上只有家人才懂的话："壮壮，姑且20岁之前我们还这么叫你，爸爸妈妈永远爱你。"
生日快乐歌再度高潮，满屏星光粒子绽放。

**为什么优先**: 这是整个游戏的情感终点，留下最深刻的记忆。

**独立测试**: 直接触发第19颗星逻辑，验证彩蛋动画和音效完整播放。

**验收场景**:

1. **给定** 用户点击第19颗星，**当** 彩蛋触发，**则** 全屏出现光晕+家庭照片+专属文字+粒子动画，持续至少5秒。
2. **给定** 终幕彩蛋结束，**当** 动画完成，**则** 出现"重新旅行"选项，允许用户重新体验完整旅程。

---

### 边界情况

- 照片加载慢时，展示优雅的骨架屏或星光占位动画，不出现破图。
- 用户刷新页面时，从头开始旅程（无需保存进度，体验简单纯粹）。
- 手机横屏时，布局自适应，核心内容不裁剪。
- 用户快速连续点击切换场景时，动画不叠加或错乱。
- 浏览器不支持某些特效时，优雅降级为静态展示，核心文字和照片仍完整可见。

## 需求规格 *(必填)*

### 功能需求

- **FR-001**: 系统必须在手机浏览器（iOS Safari、Android Chrome）中完整可用，无需安装 App。
- **FR-002**: 系统必须通过 GitHub Pages 部署，生成可通过公网访问的 HTTPS 链接。
- **FR-003**: 系统必须支持8个记忆场景的顺序浏览，且支持前后自由切换。
- **FR-004**: 系统必须在每个场景展示对应年龄段的精选照片（各年龄段照片已存在于本地目录）。
- **FR-005**: 系统必须展示19颗可点击的星星，每颗对应一句独立祝福文字。
- **FR-006**: 系统必须提供背景音乐（生日快乐歌）的播放控制（播放/暂停）。
- **FR-007**: 系统必须在第19颗星点击时触发终幕彩蛋动画序列。
- **FR-008**: 系统必须在纯静态环境下运行（无后端服务器、无数据库），所有资源内嵌于项目。
- **FR-009**: 系统必须提供"重新旅行"功能，允许用户从头开始体验。
- **FR-010**: 系统必须针对蓝紫色、樱花、星空、海洋意象设计视觉主题。

### 关键实体

- **记忆场景（MemoryScene）**: 包含年龄标签、爸爸的话、对应照片列表、彩蛋动画类型
- **祝福星（BlessingStar）**: 包含序号（1-19）、祝福文字、是否为终幕彩蛋星（第19颗特殊处理）
- **照片资源（PhotoAsset）**: 从源目录筛选后压缩优化、内嵌于项目的各年龄段代表性照片

## 假设与约束

### 假设

- 照片源目录（189张，0-18岁各阶段）中各场景均有可用照片；每场景精选1-3张代表性图片。
- 背景音乐使用免版权生日快乐歌音频（mp3格式，需家长自行提供或使用公版）。
- GitHub Pages 仓库由家长自行创建，Claude 协助配置部署。
- 祝福文字内容固定，游戏本身无需编辑界面。

### 约束

- 纯静态网页（HTML + CSS + JavaScript），不依赖任何后端服务。
- 所有资源本地化，不依赖第三方 CDN（确保弱网环境可用）。
- 游戏链接仅分享给家庭成员，不作公开推广，无需考虑高并发。
